# 🧪 Тестирование синхронизации

## Подготовка

### 1. Запустите миграцию (если БД уже существует)
```bash
npm run migrate
```

### 2. Запустите сервер
```bash
npm start
```

---

## Тест 1: Регистрация в Offline режиме

### Шаги:
1. Откройте `http://localhost:3000`
2. Нажмите "Регистрация"
3. Заполните форму:
   - Имя: `Тестовый Студент`
   - Email: `test@example.com`
   - Пароль: `123456`
4. Войдите с этими данными

### Ожидаемый результат:
- Пользователь создан с `sync_status = 'local'`
- В настройках (Настройки → Синхронизация) показан статус "Локальный"

### Проверка в консоли:
```javascript
// В DevTools консоли
App.currentUser
// Должен показать sync_status: "local", created_offline: true
```

---

## Тест 2: Страница настроек

### Шаги:
1. Войдите в систему
2. В боковом меню нажмите "Настройки" (иконка шестерёнки)

### Ожидаемый результат:
- Видна карточка "Профиль" с именем, email, ролью
- Видна карточка "Google аккаунт" с кнопкой "Привязать Google"
- Видна карточка "Синхронизация" со статусом и кнопкой

---

## Тест 3: Привязка Google (требуется Online режим)

### Требования:
- В `.env` настроены `GOOGLE_CLIENT_ID` и `GOOGLE_CLIENT_SECRET`
- Сервер доступен по URL, указанному в `GOOGLE_CALLBACK_URL`

### Шаги:
1. Войдите обычным способом (email/пароль)
2. Перейдите в "Настройки"
3. Нажмите "Привязать Google"
4. Пройдите авторизацию в Google
5. После редиректа увидите уведомление об успехе

### Ожидаемый результат:
- Появится уведомление "Google успешно привязан"
- В настройках Google показан как привязанный
- Теперь можно входить и через Google, и через пароль

### Проверка:
1. Выйдите из системы
2. Нажмите "Войти через Google"
3. Должны войти в тот же аккаунт

---

## Тест 4: Вход через Google (новый пользователь)

### Шаги:
1. Выйдите из системы (если залогинены)
2. Нажмите "Войти через Google"
3. Выберите Google аккаунт, который НЕ привязан

### Ожидаемый результат:
- Создастся новый аккаунт с ролью "студент"
- `password_hash` будет NULL (вход только через Google)
- `sync_status = 'synced'`

---

## Тест 5: Синхронизация пользователя

### Требования:
- Настроен `SYNC_URL` в `.env`
- Центральный сервер запущен и доступен

### Шаги:
1. Зарегистрируйтесь в offline режиме
2. Перейдите в "Настройки"
3. Нажмите "Синхронизировать"

### Возможные результаты:
- ✅ **Успех**: статус изменится на "Синхронизирован"
- ⚠️ **Конфликт**: появится диалог выбора (если email занят)
- ❌ **Ошибка**: "Синхронизация не настроена" (если нет SYNC_URL)

---

## Тест 6: Автоматический диалог синхронизации

### Условия:
- Пользователь создан offline (`created_offline = 1`)
- Режим сменился на online
- `sync_status = 'local'`

### Ожидаемый результат:
При входе появится модальное окно с опциями:
- Синхронизировать сейчас
- Привязать Google
- Пропустить

## Проверка через API (Postman/curl)

### Статус синхронизации:
```bash
curl http://localhost:3000/api/sync/status \
  -H "Cookie: connect.sid=YOUR_SESSION_COOKIE"
```

### Статус пользователя:
```bash
curl http://localhost:3000/api/sync/user-status \
  -H "Cookie: connect.sid=YOUR_SESSION_COOKIE"
```

## Проверка базы данных

### Откройте БД и проверьте таблицу users:
```sql
SELECT id, email, name, google_id, sync_status, created_offline, remote_id 
FROM users;
```

### Ожидаемые значения:

| Сценарий | google_id | password_hash | sync_status |
|----------|-----------|---------------|-------------|
| Offline регистрация | NULL | ✓ хеш | local |
| После привязки Google | ✓ id | ✓ хеш | synced |
| Только Google | ✓ id | NULL | synced |
| После синхронизации | любой | любой | synced |

## Возможные проблемы

### Кнопка Google не появляется
- Проверьте, настроен ли Google OAuth в `.env`
- Проверьте `/api/app-mode` - должен вернуть `google_enabled: true`

### Ошибка при привязке Google
- Проверьте `GOOGLE_CALLBACK_URL` - должен совпадать с настройками в Google Console

### Синхронизация не работает
- Проверьте `SYNC_URL` в `.env`
- Проверьте доступность центрального сервера
- Проверьте `SYNC_API_KEY` на обоих серверах
